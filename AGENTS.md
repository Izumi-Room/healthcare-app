# VitaTree / healthcare-app

## Stack

Flutter 3.x / Dart 3.x — Riverpod (`StateNotifier`), GoRouter (`ShellRoute`), Hive (local), Firebase Auth + Realtime Database, `fl_chart`, `google_fonts` (Nunito), Lottie, `flutter_local_notifications`, `intl` (id locale).

## Architecture

```
lib/
  main.dart                   — entrypoint (bootstrap Hive, Firebase, notifications, date locale)
  firebase_options.dart       — generated by FlutterFire CLI, not hand-edited
  main_scaffold.dart          — bottom nav shell (5 tabs: Pohon, Quest, Tidur, Statistik, Refleksi)
  core/
    theme.dart                — AppColors, AppDimens, AppAnimations, AppTheme (Nunito, Material3)
    router.dart               — GoRouter: auth redirect guard, ShellRoute for tab nav
  features/
    auth/                     — LoginScreen, RegisterScreen, auth_provider (FirebaseAuth)
    home/                     — HomeScreen, tree_widget, leaf_particle, health_score_provider
    quest/                    — QuestScreen, quest_provider, quest_timer_sheet
    sleep/                    — SleepScreen, sleep_provider, 9 widget files
    stats/                    — StatsScreen, stats_provider, 8 widget files
    reflection/               — ReflectionScreen, reflection_flow_screen, 9 step widgets, trigger provider
  services/
    firebase_database_service.dart   — singleton, offline persistence 100MB, CRUD for all nodes
    notification_service.dart        — flutter_local_notifications wrapper
  models/                     — FirebaseUserProfile, FirebaseTreeState, FirebaseStats, etc.
  shared/widgets/             — mascot_loading, mascot_helper, tactile_components
```

## Key facts

- **Firebase project**: `healtcare-dd7ea` — Realtime Database (`/users/{uid}/...`, `/quests/{uid}`, `/reflections/{uid}`, `/sleep/{uid}`)
- **Auth guard in router**: `/login` and `/register` are unauthenticated; all other routes require auth. Redirect happens in `GoRouter.redirect`.
- **Date locale**: `'id'` via `initializeDateFormatting('id', null)` — must be awaited in bootstrap.
- **Hive init**: `Hive.initFlutter()` before any box operations. Test creates a temp dir.
- **Tree assets**: `assets/trees/tree_lv1.png` through `tree_lv10.png`. No levels 0 or 11+.
- **Mascot assets**: `assets/mascot/` (wink, think, loading, excited PNGs), `assets/lottie/mascot_leaf.json`.
- **Level-up formula**: XP threshold = `level * 500`. Excess XP carries over. Applied in `rewardUserReward` via transaction.
- **Navigation**: `ShellRoute` with `NoTransitionPage` for all 5 tab routes. Bottom nav uses `NavigationBar` with `WidgetStateProperty`.
- **Testing**: `setupFirebaseCoreMocks()` required before `Firebase.initializeApp()`. `GoogleFonts.config.allowRuntimeFetching = false` to avoid HTTP in tests. Auth state must be injected via `ProviderScope` overrides.

## Conventions

- Feature-first directory layout under `lib/features/`.
- State: `StateNotifier` + Riverpod provider per feature.
- `AppTheme.light()` only — no dark theme.
- `NoTransitionPage` for all ShellRoute children (no slide animation between tabs).
- Firebase Realtime Database paths match model names: `users/$uid/profile`, `users/$uid/tree`, `users/$uid/settings`, `users/$uid/stats`, `quests/$uid`, `reflections/$uid`, `sleep/$uid`, `achievements/$uid`.
- `ServerValue.increment` used for atomic counter updates.
- `keepSynced(true)` on streamed refs for offline support.
